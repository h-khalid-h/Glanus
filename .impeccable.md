# Glanus Design Context

## Design Context

### Users
IT administrators, DevOps engineers, and operations teams managing hardware/software inventories at the enterprise level. They live in dashboards all day — they need clarity, speed, and confidence. Not delight for delight's sake, but precision that builds trust.

### Brand Personality
**Precise · Powerful · Professional**
The interface should feel like a tool built by experts for experts — not a consumer SaaS. Every element earns its place. Data is surfaced clearly. Hierarchy is immediate. The brand exudes quiet competence.

### Aesthetic Direction
- **Theme**: Auto (light + dark, respects system preference)
- **Color**: Teal primary (`#00E5C8`, `hsl(168 100% 45%)`), navy dark backgrounds, amber/oracle for warnings, green/reflex for healthy states
- **Typography**: Refined, modern — sharp hierarchy between headings and data; never decorative for its own sake
- **Visual tone**: Command-center clarity. Data-dense without feeling cluttered. Structured, not boxy. Whitespace is intentional.
- **Anti-references**: No generic SaaS clones (no Linear/Notion feel), no glassmorphism clichés, no gradient-text headings, no bounce animations, no consumer app playfulness

### Design Principles
1. **Clarity above all** — if information isn't instantly readable, rethink the layout
2. **Hierarchy through contrast** — use weight, color, and size to guide the eye, not borders and boxes
3. **Status at a glance** — operational health must be visible without interaction
4. **Professional restraint** — motion and decoration are purposeful, never decorative
5. **System coherence** — use Glanus design tokens (nerve/cortex/oracle/reflex/health-*) consistently; never introduce ad-hoc colors
